What is HTML?

HTML (HyperText Markup Language) is the skeleton of every webpage. It defines the structure and meaning of web content using elements called tags.

🌐 How a Webpage Works

Every website you visit is built from three core technologies:

Language Role Analogy
HTML Structure & content Skeleton / blueprint
CSS Visual styling Paint & decoration
JavaScript Interactivity Muscles & movement

🏷️ Your First HTML Tag

An HTML element has an opening tag, content, and a closing tag:

html 
<!-- Anatomy of an element -->
<p>Hello, World!</p>

<!-- Some tags are self-closing (void elements) -->
<br>
<hr>
<img src="photo.jpg" alt="A photo">
💡 Opening tag = <tagname>  |  Closing tag = </tagname>  |  Void = no closing tag needed

Document Structure

Every HTML file follows a standard template. Understanding this skeleton is the foundation of everything you build.

📄 The Boilerplate

This is the minimum required structure for any valid HTML page:

html 
<!-- Tells the browser: "This is HTML5" -->
<!DOCTYPE html>

<!-- Root element, lang helps screen readers -->
<html lang="en">

  <head>
    <!-- Character encoding: always UTF-8 -->
    <meta charset="UTF-8">
    <!-- Responsive on mobile devices -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <!-- Title shown in browser tab -->
    <title>My Page</title>
  </head>

  <body>
    <!-- Everything visible goes here -->
    <h1>Hello, World!</h1>
  </body>

</html>

🧠 Head vs Body

Section Contains Visible?
<head> Metadata, title, CSS links, scripts No
<body> All visible content Yes
✅ Always include <!DOCTYPE html> as the very first line — it enables Standards Mode in all browsers.

Text & Headings

HTML gives you rich tools for structuring text — from headings and paragraphs to bold, italic, and more.

📰 Headings h1–h6

HTML has six heading levels. Use them in order — they communicate hierarchy, not just size.

html 
<h1>Page Title (use once)</h1>
<h2>Section Heading</h2>
<h3>Sub-section</h3>
<h4>Sub-sub-section</h4>
<h5>Minor heading</h5>
<h6>Smallest heading</h6>
⚠️ Only use <h1> once per page. Skipping levels (e.g. h1 → h3) hurts accessibility.

📝 Text Formatting Tags

Tag Result Semantic meaning
<strong> Bold Important content
<em> Italic Emphasized content
<mark> Highlighted Relevant/highlighted
<code> code Inline code
<del> Deleted Removed content
<sup> / <sub> X2 / H2O Superscript/subscript

Links & Images

Links connect the web together. Images make it visual. These two tags are used on virtually every page.

🔗 The Anchor Tag <a>

html 
<!-- External link (opens in new tab) -->
<a href="https://example.com" target="_blank" rel="noopener">
  Visit Example
</a>

<!-- Internal link to another page -->
<a href="about.html">About Us</a>

<!-- Link to a section on the same page -->
<a href="#section-id">Jump to Section</a>

<!-- Email link -->
<a href="mailto:hello@example.com">Email Me</a>
✅ Always add rel="noopener noreferrer" with target="_blank" to prevent security issues.

🖼️ The Image Tag <img>

html 
<!-- The alt attribute is REQUIRED for accessibility -->
<img 
  src="dog.jpg"
  alt="A golden retriever on a sunny beach"
  width="400"
  height="300"
>

<!-- Use a URL for remote images -->
<img src="https://placekitten.com/200/150" alt="A kitten">
⚠️ Never leave alt empty unless the image is purely decorative (alt=""). Screen readers depend on it.

Lists & Tables

Lists organize items. Tables display structured data. Both are everyday HTML essentials.

📌 Three Types of Lists

html 
<!-- Unordered (bullet) list -->
<ul>
  <li>Apples</li>
  <li>Bananas</li>
</ul>

<!-- Ordered (numbered) list -->
<ol>
  <li>First step</li>
  <li>Second step</li>
</ol>

<!-- Description list (term + definition) -->
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
</dl>

📊 Tables

html 
<table>
  <thead>
    <tr>
      <th>Name</th>  <th>Score</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>  <td>95</td>
    </tr>
    <tr>
      <td>Bob</td>    <td>87</td>
    </tr>
  </tbody>
</table>
💡 Tables are for data, not layout. Never use a table to arrange your page's visual structure.

Forms & Inputs

Forms are how users interact with websites — logins, search bars, contact pages, checkouts. Mastering forms is essential.

🧰 Essential Form Elements

html 
<form action="/submit" method="post">

  <!-- Label + input pair (always link them!) -->
  <label for="name">Full Name</label>
  <input type="text" id="name" name="name" placeholder="Alice Smith" required>

  <label for="email">Email</label>
  <input type="email" id="email" name="email" required>

  <!-- Dropdown -->
  <label for="role">Role</label>
  <select id="role" name="role">
    <option value="student">Student</option>
    <option value="dev">Developer</option>
  </select>

  <!-- Multiline text -->
  <label for="msg">Message</label>
  <textarea id="msg" name="msg" rows="4"></textarea>

  <button type="submit">Send</button>

</form>

🧱 Lesson 7 of 8

Semantic HTML5

Semantic tags give meaning to your content — helping browsers, search engines, and screen readers understand the page structure.

🏛️ Page Structure Tags

html 
<header>
  <nav>Site navigation links</nav>
</header>

<main>
  <article>
    <h2>Blog Post Title</h2>
    <section>Introduction section...</section>
    <section>Body section...</section>
  </article>

  <aside>Related links / sidebar</aside>
</main>

<footer>Copyright & contact info</footer>

📖 Semantic vs Non-Semantic

Avoid Use Instead Why
<div id="header"> <header> Communicates role
<div id="nav"> <nav> Screen readers use it
<div id="content"> <main> Only one per page
<b> <strong> Semantic importance
<i> <em> Semantic emphasis
✅ Semantic HTML improves SEO, accessibility (WCAG), and code readability — use it by default.
