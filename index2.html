🏛️ Structural Elements

Semantic structural tags communicate the role of each region to browsers and assistive technologies.

Semantic structural HTML5 tags
Tag Role Notes
<header> Page or section header May contain logo, nav, search
<nav> Navigation links Use aria-label when multiple navs exist
<main> Primary content One per page, skip-link target
<article> Self-contained content Blog post, card, comment
<section> Thematic grouping Should have a heading
<aside> Tangentially related Sidebar, pull-quote, ad
<footer> Closing info Author, copyright, links
<address> Contact info For the nearest article or body
Source: HTML Living Standard (WHATWG)

🖼️ Media Elements

Figure & Figcaption

Responsive Images <picture>

The <picture> element lets you serve different images based on viewport size or format support: 

<picture>
  <source media="(min-width: 800px)" srcset="large.webp" type="image/webp">
  <source media="(min-width: 800px)" srcset="large.jpg">
  <img src="small.jpg" alt="Descriptive alt text" loading="lazy">
</picture>

Audio <audio>

The <audio> element embeds sound with native browser controls: 

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

Video <video>

<video controls width="640" poster="thumbnail.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  <track kind="subtitles" src="subs.vtt" srclang="en" label="English">
  Your browser does not support the video element.
</video>

⚡ Interactive Elements

Details & Summary (native accordion)

What is semantic HTML?

Semantic HTML means using the correct HTML element for its intended purpose — <nav> for navigation, <article> for standalone content, etc. — rather than generic <div> tags.

Why does accessibility matter?

Over 1 billion people live with a disability. Accessible HTML ensures screen readers, keyboard users, and other assistive technologies can use your content. It also improves SEO and overall code quality.

What is the difference between <strong> and <b>?

<strong> conveys semantic importance (screen readers may announce it). <b> is purely stylistic bold with no extra meaning. Prefer <strong>.
