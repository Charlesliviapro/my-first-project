Variables & let / const

Variables are named containers for storing data. In modern JavaScript you use let and const — forget var.

📦 let vs const vs var

Keyword Re-assignable? Block scope? Use when…
const No Yes Value won't change (default choice)
let Yes Yes Value will be reassigned
var Yes No Avoid — legacy code only
⚠️ Default to const. Only switch to let when you know the value will change.

📏 Naming Rules

javascript 
// ✅ Good — camelCase is the convention
const firstName = "Alice";
const totalScore = 100;
const isLoggedIn = true;

// ❌ Bad — these cause errors or confusion
// const 1name = "bad";   ← cannot start with a number
// const my-name = "bad"; ← hyphens not allowed
// const class = "bad";   ← reserved keyword
💡 Use camelCase for variables: myVariableName. Use UPPER_SNAKE_CASE for true constants: MAX_SIZE = 100.

⚖️ Comparison & Equality

javascript 
// === strict equality (value AND type)
console.log(5 === 5);    // true
console.log(5 === "5");  // false ← different types

// == loose equality (coerces types — avoid!)
console.log(5 == "5");   // true ← surprising!

// Other comparisons
console.log(10 > 5);     // true
console.log(3 !== 4);    // true
⚠️ Always use === (triple equals), never ==. Triple equals checks both value and type.

Control Flow

Control flow lets your program make decisions — running different code depending on conditions.

🔀 if / else if / else

javascript 
const score = 72;

if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 70) {
  console.log("Grade: B");
} else if (score >= 50) {
  console.log("Grade: C");
} else {
  console.log("Grade: F");
}

❓ Ternary & Switch

javascript 
// Ternary: condition ? valueIfTrue : valueIfFalse
const age = 20;
const status = age >= 18 ? "adult" : "minor";
console.log(status); // "adult"

// Switch — great for many exact matches
const day = "Monday";
switch (day) {
  case "Monday":
    console.log("Start of the week"); break;
  case "Friday":
    console.log("Almost weekend!"); break;
  default:
    console.log("Midweek");
}

Loops

Loops repeat a block of code — essential for processing lists, counting, and automating repetitive tasks.

🔁 for Loop

javascript 
// for (start; condition; step)
for (let i = 1; i <= 5; i++) {
  console.log("Count:", i);
}
// Prints: Count: 1, Count: 2 ... Count: 5

🔂 while & for...of

javascript 
// while — runs while condition is true
let n = 1;
while (n <= 3) {
  console.log(n);
  n++;
}

// for...of — loop over arrays (cleaner!)
const fruits = ["apple", "banana", "cherry"];
for (const fruit of fruits) {
  console.log(fruit);
}
✅ Use for...of when looping over arrays — it's cleaner than a traditional for loop.

Functions

Functions are reusable blocks of code. They are the single most important concept in JavaScript.

🎯 Three Ways to Write Functions

javascript 
// 1. Function declaration (hoisted)
function greet(name) {
  return `Hello, ${name}!`;
}

// 2. Function expression
const double = function(n) {
  return n * 2;
};

// 3. Arrow function (modern, concise)
const square = (n) => n * n;

console.log(greet("Alice"));  // Hello, Alice!
console.log(double(7));       // 14
console.log(square(4));       // 16

📥 Parameters & Default Values

javascript 
// Default parameter values
function introduce(name, role = "Developer") {
  return `${name} is a ${role}`;
}
console.log(introduce("Alice"));           // Alice is a Developer
console.log(introduce("Bob", "Designer")); // Bob is a Designer

Arrays

Arrays store ordered lists of values. They come with powerful built-in methods that you'll use in nearly every program.

📋 Array Basics

javascript 
const fruits = ["apple", "banana", "cherry"];

console.log(fruits[0]);        // "apple" (index starts at 0)
console.log(fruits.length);    // 3

fruits.push("mango");          // add to end
fruits.pop();                   // remove from end
fruits.unshift("lemon");      // add to start
fruits.shift();                 // remove from start

🔧 The Big Three Array Methods

javascript 
const nums = [1, 2, 3, 4, 5];

// map — transform every element
const doubled = nums.map(n => n * 2);
console.log(doubled); // [2, 4, 6, 8, 10]

// filter — keep only matching elements
const evens = nums.filter(n => n % 2 === 0);
console.log(evens);   // [2, 4]

// reduce — collapse to a single value
const sum = nums.reduce((acc, n) => acc + n, 0);
console.log(sum);     // 15
✅ map, filter, and reduce are the most important array methods. Master these and you can do almost anything with data.

Objects

Objects store data as key-value pairs. They model real-world things — a user, a product, a blog post.

🗂️ Creating & Accessing Objects

javascript 
const user = {
  name: "Alice",
  age:  28,
  role: "Developer",
  isActive: true
};

// Dot notation (most common)
console.log(user.name);       // "Alice"

// Bracket notation (for dynamic keys)
console.log(user["role"]);    // "Developer"

// Add or update a property
user.email = "alice@example.com";

// Destructuring — extract into variables
const { name, age } = user;
console.log(name, age);       // "Alice" 28

🔑 Looping & Spread

javascript 
const config = { theme: "dark", lang: "en", size: 14 };

// for...in — loop over object keys
for (const key in config) {
  console.log(`${key}: ${config[key]}`);
}

// Spread — copy/merge objects
const updated = { ...config, size: 16, color: "blue" };
console.log(updated);
